# Design: v0.3.0 Public Release

**Date:** 2026-05-06  
**Status:** Approved  
**Target version:** 0.3.0

## Overview

First public release of `local-addon-zip-launcher`. Ships to GitHub as a public repository with GitHub Releases for distribution and GitHub Pages for documentation. No Local addon marketplace submission. Follows the established `local-addon-laravel` pattern exactly.

**Approach:** Phased ŌĆö four sequential phases, each independently committable:
1. Code fixes + TypeScript migration
2. Documentation + GitHub Pages
3. CI/CD workflows
4. Cut v0.3.0 GitHub Release

### `docs/DEVELOPER_GUIDE.md`

- Prerequisites: Node 18+, Local installed
- Setup: `git clone` ŌåÆ `npm install` ŌåÆ `npm run install-addon` ŌåÆ restart Local
- Development: `npm run watch` (auto-recompile) + restart Local to pick up changes
- Tests: `npm test` / `npm run test:coverage`
- Architecture overview: renderer capture ŌåÆ main IPC handler ŌåÆ zip-analyzer ŌåÆ collision detection ŌåÆ demo content import
- How to add a new zip detection type

### `docs/TROUBLESHOOTING.md`

| Symptom | Cause | Fix |
|---|---|---|
| "Not a theme or plugin zip" | No `Theme Name:`/`Plugin Name:` header found | Verify the zip contains a `style.css` or PHP file with a WordPress header at depth Ōēż 2 |
| Collision dialog ŌåÆ nothing | Site not running, start failed | Start the site manually from Local, then drop again |
| Demo content didn't import | `wordpress-importer` install failed (network) | Install plugin manually, import XML via WP Admin ŌåÆ Tools ŌåÆ Import |
| Drop overlay stuck | Browser event state issue | Restart Local |

### Known limitations (in USER_GUIDE)

- `.zip` only ŌĆö `.tar.gz` not supported
- One zip per drop ŌĆö first zip wins if multiple are dropped simultaneously
- Manual install ŌĆö no Local addon marketplace listing
- Admin password is randomly generated ŌĆö Local auto-logs you in
- Demo content import requires internet access (to install `wordpress-importer`)
- Theme/plugin detection requires header at depth Ōēż 2 in the zip

## Phase 3: CI/CD Workflows

### `.github/workflows/ci.yml`

Triggers: push to `main`/`develop`, PRs to `main`, `workflow_call`.

Four parallel jobs: `lint`, `typecheck`, `test` (with Codecov), `build` (uploads `lib/` artifact).

### `.github/workflows/release.yml`

Triggers: `v*` tags.

1. Calls `ci.yml` (reuse)
2. `npm ci` + `npm run build` + `npm pack`
3. `softprops/action-gh-release@v1` ŌĆö attaches `.tgz`, auto-generates release notes from conventional commits, marks prerelease if tag contains `-alpha` or `-beta`

### `.github/workflows/docs.yml`

Triggers: push to `main` when `docs/**`, `README.md`, `index.md`, or `_config.yml` change; manual dispatch.

Jekyll build ŌåÆ deploy to GitHub Pages (same action chain as `local-addon-laravel`).

## What ships in v0.3.0

- Drop any `.zip` anywhere in Local ŌåÆ theme/plugin auto-detected ŌåÆ site created
- Collision detection with native dialog (Update existing / Create new / Cancel)
- Auto-start stopped site on update
- Demo content (WXR) auto-import with progress bar
- WordPress.org PHPDoc header format support
- TypeScript source, CI green, GH Pages docs live

## Not in v0.3.0 (future)

- Local addon marketplace listing
- Multiple zip support
- `.tar.gz` support
- Custom WP/PHP version selection at drop time
- Automated integration tests (full site creation flow)
